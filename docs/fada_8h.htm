fada_TSample  sample_type,
unsigned int  sample_rate,
unsigned int  channels 
)

Bind initial audio data and audio information to the manager.

If successful, the manager will be ready to use for analysis.

Parameters
mThe manager to bind.
dataPointer to the audio data.
sample_countThe number of samples stored in data.
copy_dataCopy the data being bound to the manager. Set to FADA_FALSE to reference the data instead.
sample_typeThe sample type of audio data.
sample_rateThe sample rate of the audio data in Hertz. For example, 44100.
channelsThe number of audio channels. For example, 1 would be mono audio, 2 would be stereo audio, etc.
Returns
Returns one of the following error signals:
  • FADA_ERROR_INVALID_CHANNEL
  • FADA_ERROR_INVALID_SAMPLE_RATE
  • FADA_ERROR_INVALID_SIZE
  • FADA_ERROR_INVALID_TYPE
  • FADA_ERROR_MANAGER_NOT_READY
  • FADA_ERROR_NO_DATA
  • FADA_ERROR_NOT_ENOUGH_MEMORY
  • FADA_ERROR_NOT_MULTIPLE_OF_CHANNELS
  • FADA_ERROR_SUCCESS

FADA_API fada_Boolean fada_continue ( fada_Manager *  m,
long  offset_frames 
)

Advance the analysis window position ahead.

If offset_frames is below 0 or FADA_NEXT_WINDOW, it will automatically continue to the next window of samples. You may also multiply FADA_NEXT_WINDOW to skip windows. For example, fada_continue(m, FADA_NEXT_WINDOW * 2) will jump two windows ahead, stopping at the window after the next.

Parameters
mThe manager to advance.
offset_framesNumber of frames to advance. Can be negative to skip over whole windows.
Returns
Boolean indicating whether the end of audio data has not been reached. FADA_FALSE if end of audio has been reached.

FADA_API fada_FFTBuffer* fada_newfftbuffer ( fada_Pos  size_po2)

Create a new external buffer.

size_po2 must be a power of 2. If it is not a power of 2, it will use the closest power of 2 less than size_po2. FFT buffers allocate sizeof(fada_Res) * size_po2 * 2 of memory, which is twice more memory than one may expect. size_po2 only defines the size of the "useful" half of the FFT.

FADA_API fada_Error fada_preloadfftbuffer ( fada_Manager *  m)

Preallocate and use the internal FFT buffer.

If the manager doesn't have an FFT buffer already assigned to it, this function is automatically called when an FFT buffer is needed. You can manually call this function to use the internal FFT buffer during application initialization, so libfada won't need to call this automatically. Internal FFT buffer size will be the frame count of the manager's analysis window. If the window's frame count is not a power of 2, it will use the closest power of 2 less than the window's frame count.

Note
It is important to remember that this function will not work as intended if an FFT buffer is still assigned to the manager (via fada_usefftbuffer). You must unassign the buffer first: fada_usefftbuffer (m, NULL)
Parameters
mThe manager.
Returns
Returns one of the following error signals:
  • FADA_ERROR_MANAGER_NOT_READY
  • FADA_ERROR_NOT_ENOUGH_MEMORY
  • FADA_ERROR_SUCCESS
